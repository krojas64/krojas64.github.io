Testing is a simple way to test individual functions of python code
to ensure that they are outputing the proper values. This is important because
as you continue adding new methods and functions to your project, the
some of the changes made will more than likely affect the output of previously
created methods. Sometimes you might find a more efficient way to create something,
which may result in overhauling a good chunk of code. This is why having a set of tests
that check your methods everytime you rewrite and build your code is extrememly important.
It'll help you find bugs in your code, ensure that your program is running smoothly, but
most importantly, that it is creating the output the way you expect it to.
In order to create tests, you must install pytest:
pip install pytest
From here, you can make files that end with _test.py, which lets pytest know
that these files are specifically for testing. When testing a specific method, you must
import the method with the following:
from (filepath) import (methodname)
Once imported, you can create another method that will test the imported method.
There are three steps to follow: Arrange, Act, and Assert. Arranging is getting
values that are going to be used for the test. In the example below, I use a tuple
with two numbers. Then you act, which is where the result is generated from.
In this case, it is the result of using the arranged values with the Addition method.
Finally, you assert the result, and check to see if it matches with what is expected.
Act and Assert can both be done in one line, but in this case, they are separated.
