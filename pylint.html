Pylint Intro

Return to Main Page
Pylint is desgined to check your code in order to see if it meets the general
standards of how Python code should be written. If your code works well, but
looks messy and is nearly impossible to read, then you will have problems
in the future. Pylint will give you warnings on what is missing in your code
and adjustments that should be made to improve readability. In order to use Pylint, you must install it using: 
                
                pip install pylint
Afterwords you must go into the directory you want to run pylint on. From here
you can set which file you would like pylint to run on. For example, on the
following Github page,
if we wanted to test the main file, we would run: 
		
		pylint calculator/main.py
Pylint is used so the terminal knows that we are running pylint on a file, and
the second argument is used to determine the filepath of the file to run pylint on.
Once run, the terminal will check the entire file and print out a list of errors
and warnings, explaining what is causing the error. This is where adjustments can be
made to your code, and as the number of errors decreases, your overall score will
also increase, indicating that your code is closer to the standard.

General Terms to know in Python

  • Factory - A method in a class that is used to create objects, however it does not
    instantiate the object. Subclasses that inherit the object will determine what
    the object does.
  • Object - What most "things" (for lack of better term) in Python are made up of.
    Objects have properties that can determine what it is, what it should run, and how
    it is affected by other objects.
  • Class - A method that has other methods used to create and define objects. They
    handle initialization of the object, which for example, a fatory method would be used.
  • Namespace - A list of names that are currently being used to refer to a specific
    value, object, or method. Once a name has been used, that name can be called to
    get the value or method associated with it.
  • Constructor - A method used to instantiate an object. This methos specifically
    defines what variables the object has and what it can do with those variables
