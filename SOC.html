Separation of Concerns

Return to Main Page
The main idea of separation of concerns is making sure that certain
parts of code don't serve multiple functions. Code that is meant to
calculate or determine different values should stay together, and not
spread around wherever it may be convenient. For example, in this GitHub repo we can see that the
caluclations and tests are within their respective directories. Looking
into each directory, we can see that each part is separated based on what
purpose they serve. In calc, we have an operations directory, where the
calculation base class and other operations are kept. They all serve a
similar purpose, which is why they are kept in this order.

Meanwhile the tests are also separated based on the different operation they test,
but again, they are kept in a similar location.
If we had tests running within the operation files, this would violate the
single responsibility principle which dictates that parts of code should
mainly serve a single purpose.
